## Process notes

- **Planner caught the rename-diff subtlety**: PR-32-D03 looked like a one-line predicate flip. Planner traced `BaboonComparator.scala:247-254` and identified that rename diffs are keyed by OLD type ID (`prevIds` not `nextIds`), so the corrected formula is `diffIds.diff(nextIds).diff(prevIds)` — the additional `.diff(prevIds)` excludes legitimate rename entries from being flagged as orphans. Without that correction, the matrix would have broken on every model with renames. Lesson: when planning predicate inversions, trace the source of the underlying set, not just the predicate side.
- **PR-40 first-pass was fundamentally wrong**: `.git` walk-up resolved to repo root, but the fixture root is at `<repoRoot>/target/test-{regular,wrapped}/`. The original M15 fix to T6_D{1,2} had been incidentally regressed — tests went from passing to silently skipping, which the executor's verification missed because skip is reported as pass in dart's test output. Adversarial review caught it via empirical filesystem inspection. The redesign uses a sentinel (directory containing both `target/` and a `*-stub` sibling) plus a bootstrap-time loud-fail assertion.
- **Reviewer feedback shapes implementations heavily**: across M16 the reviewer caught five major or moderate first-pass bugs (PR-36 incomplete rename, PR-38 round-trip-masquerading-as-decoder-test, PR-39 same indent bug in Java/Kotlin, PR-40 D01-D03 fundamental anchor wrong) — all surfaced via adversarial review and fixed in second pass. This justifies the cost of the review step every time.
- **Subagent stall on PR-37**: the opus-class executor for the largest PR stalled on `mdl` progress monitoring without returning a structured report. The actual edits across 11 files were correct; orchestrator verified via the cross-language compat matrix. Lesson: large PRs may benefit from instructing the executor to return as soon as edits are complete and let the orchestrator handle verification.
