## Locked architectural decisions (Q1–Q4 + additional)

The following decisions were locked during M30 and are binding for all future
work touching the doc pipeline:

**Q1 (template propagation)**: When a template-instantiation alias `type Y = X[T]`
has its own `/** … */` prefix doc and the template `X` also has a type-level doc,
the instantiated type carries the **concatenation** `alias-doc + "\n\n" + template-doc`
as merged doc text. Implemented in `TemplateInstantiator.mergeAliasAndTemplateMeta`.

**Q2 (Python field docs)**: Python field docs are folded into the class-level PEP-257
docstring as a Google/Sphinx-style `Attributes:` section rather than appearing as
standalone `#:` annotations or field-level docstrings. This matches idiomatic Python
tooling expectations.

**Q3 (stacked prefix docs)**: Two `/** … */` blocks back-to-back with no intervening
declaration produce `ParserIssue.StackedDocComments` at the second block's position.
No accumulation semantics. Users must merge into one block or interpose a declaration.

**Q4 (cleanup policy)**: Doc cleanup (common-prefix `\s*\*?\s*` strip; content vs.
separator-line distinction; empty-doc drop) runs once at typer stage in `DocFormat.scala`.
Backends receive `Docs.cleaned` and MUST NOT re-clean. Each backend applies only its own
per-language escaping (XML for C#, `"""` for Python/GraphQL, HTML for Java).

**BaboonWhitespace pivot (PR-30.2)**: The parser uses `BaboonWhitespace` (a doc-aware
`Whitespace` implicit) everywhere instead of `ScalaWhitespace`. `BaboonWhitespace` is
behaviourally identical to `ScalaWhitespace` for non-doc-bearing code but stops at `/**`
openers and `//!` markers, leaving them for `DefDocs` rules. Any future parser site MUST
use `BaboonWhitespace`, NOT `fastparse.ScalaWhitespace`.

**Layered drop**: The parser captures docs uniformly at every `withMeta` site. The typer
drops docs at non-position carriers (enum values, ADT inheritance arms, namespaces,
foreign per-language entries) per spec §3.2/§9. Backends never see docs at non-position
carriers.

**Schema-id digest stability**: `BaboonEnquiries.shallowId` uses explicit structural
projection (`d.fields.map { f => s"${f.name}:${wrap(f.tpe)}" }`) that excludes
`Field.docs` and `MethodDef.docs`. Any future extension to the digest computation MUST
verify empirically that no existing fixture md5 churns.

**ADT-arm doc gap (PR-30.4b)**: `DefAdt.adtMember` originally double-consumed
`prefixDocs` via nested `withMeta` calls. Fixed in PR-30.4b by removing the outer
`withMeta` wrapper; inner `dtoEnclosed`/`contractEnclosed`'s `meta.member` now captures
the doc directly. Pattern applies to any future parser rule that wraps an enclosed
definition in a `meta.withMeta` outer layer.

## Constraints future work must respect

1. **Backends consume `Docs.cleaned` directly.** Do NOT re-clean in backends. Per-backend
   escaping applies only to the cleaned text, not to the raw doc.

2. **Layered drop is at the typer.** Backends never see docs at non-position carriers
   (enum values, ADT inheritance arms, namespace openers, foreign per-language entries).
   Any future backend MUST follow this invariant.

3. **Schema-id digest stability is structural by construction.** Any new typed-model field
   MUST be explicitly excluded from `BaboonEnquiries.shallowId` projection if it should
   not affect wire-format compatibility. Verify with `git diff --stat
   baboon-compiler/src/test/resources/` after the build.

4. **`BaboonWhitespace` is the canonical parser whitespace.** Any new parser rule MUST use
   `BaboonWhitespace`, NOT `fastparse.ScalaWhitespace`. Violation will silently swallow
   doc comments before any `withMeta`-anchored rule can fire.

5. **ADT-arm doc pattern:** Any new parser rule that wraps an enclosed definition in an
   outer `withMeta` and the enclosed rule itself calls `meta.member` MUST flatten the two
   layers (PR-30.4b pattern) to avoid double-consuming `prefixDocs`. Review all new
   `withMeta`-in-`withMeta` sites.

6. **M30 fixture naming.** The `m30-ok` fixture uses `M30`-prefixed type names to avoid
   short-name conflicts with `m30-sc-docs` (`DocItem`, `DocSeverity`, etc.) that would
   confuse the LSP hover resolver. New fixtures in the same workspace MUST use namespaced
   or otherwise non-clashing type names.

7. **Q1–Q4 locks are binding.** See above. In particular, Q3 (stacked docs rejected) and
   Q4 (single cleanup pass at typer stage) must not be relaxed without a new user
   decision.

8. **PR-30.4 template holds for future backends.** Any new backend MUST add a `renderDocs`
   helper + `prependDocs` wiring at type/field/method level in its `*DefnTranslator`,
   apply per-language escaping, and include at least 5 emission tests covering the
   standard doc positions. The `m30-sc-docs` fixture is the canonical shared fixture for
   per-backend emission tests; `m30-ok` is the comprehensive cross-language smoke fixture.
