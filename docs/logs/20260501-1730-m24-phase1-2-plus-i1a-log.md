## Cross-cutting design decisions

1. **PR-F discipline established:** every backend's malformed map-key path must throw a typed `BaboonCodecException`-equivalent with message starting `"malformed key: <repr>"`. Replaces 8 different mechanisms. Builtin-primitive parse arms (`Integer.parseInt`, etc.) deliberately untouched ŌĆö deferred follow-up.

2. **PR-G unification:** TS map-key wire form unified to canonical `{k: v}` string-keyed-object across all map types (str, builtin non-string, user-type). Removed pre-PR `[[k,v]]` tuple-array divergence. Latent enum-key gap surfaced and fixed during unification.

3. **PR-H auto-routing:** `RsBaboonTranslator` emits both `lib.rs` and `mod.rs` aggregators with content dynamically derived from emit set. Adding new top-level Rust modules no longer requires hand-editing `test/conv-test-rs/src/lib.rs`. Side benefit: orphaned `baboon_service_wiring.rs` is now correctly auto-routed.

4. **PR-I.1a reference pattern (CRITICAL ŌĆö propagates to 8 sub-PRs):**
   - Trait + companion `object` with `@volatile var _instance` + `register/instance`.
   - Stringy decl (allowlist match): identity DefaultImpl auto-emitted.
   - Non-stringy decl: stub-throw DefaultImpl with FQN-bearing diagnostic.
   - BaboonRef-mapped: skip emission entirely.
   - Decoder catches `Exception` (NOT `Throwable`) ŌĆö excludes `Error` and `ControlThrowable`.
   - Encoder uses explicit `Custom` match ŌĆö never wildcarded.
   - Test asserts byte-identity (`noSpaces == "..."`), NOT Circe `Json` structural equality.
   - Diagnostic message includes FQN so it's copy-pasteable as an import path.

   Pattern-propagation guidance baked into `defects.md` PR-I-D01..D09 entries ŌĆö required reading for any sub-PR executor.

## Worktree friction (recurring across Phase 1)

Phase 1 used parallel worktrees (wt-pr-a..e). Recurring issues:
- **jgit + worktrees:** sbt-git couldn't find git objects in worktrees. Workaround: symlink `.git/objects` and `.git/config` from main repo into worktree's pseudo-`.git` dir.
- **mudyla + worktrees:** `find_project_root` requires `.git` to be a directory not a file. Workaround: temporarily replace the worktree's `.git` file with a symlink for `mdl` invocations.
- **Worktree teardown:** two worktrees couldn't be removed via `git worktree remove --force` because of the symlink workaround; required `rm -rf` + `git worktree prune`.

Phase 2 and Phase 3 sub-PRs ran in main checkout serially ŌĆö simpler, avoids the friction.
