## Cross-backend pattern propagation (PR-I family)

PR-I.1a (Scala) established the canonical pattern; 8 sub-PRs mirrored it. Key invariants documented in defects.md PR-I-D01..D09 + per-sub-PR entries:

- **Catch only "exceptions you'd handle"** (NOT Throwable/BaseException). Per-language: Java/Kotlin/Dart `Exception`; Swift `catch let e`; TS default-typed binding; Python `except Exception`; Rust serde-Result with no panic-catch.
- **Byte-identity test assertion via canonical literal**, NOT structural object equality.
- **FQN-bearing diagnostic** in stub-throw default — references the **Host** name, not the interface (PR-I.1b-D01 caught Java getting this wrong; pattern propagated).
- **Explicit `Custom(_, _)` encoder/decoder match**, never wildcarded.
- **Stringy allowlist** per-backend: see plan body §"Allowlist".
- **BaboonRef-mapped foreigns skip emission** entirely (recursion handles).

## Architecturally distinct cases

- **Rust (PR-I.3):** serde-with-adapter pattern. Per-foreign trait + `OnceLock<Box<dyn _>>` host singleton + sibling `<foreign>_as_map_key` adapter module (`serialize<S, V>` + `deserialize<'de, __De, V>`). Wrapper-around-foreign path inlines `<foreign>_keycodec()` calls in the wrapper's adapter rather than chaining through `<foreign>_as_map_key` (which is whole-map, not per-key).
- **Swift (PR-I.2):** PR-B's `mayThrow` plumbing load-bearing — Foreign-Custom decoder returns `(decExpr, true)` to trigger outer `try` propagation in `Dictionary(uniqueKeysWithValues:)` call.
- **Python (PR-I.2 round-2):** `_try_decode_key(self, fn, raw_repr)` per-codec helper method needed because Python expressions cannot contain try/except. Maintains cross-backend "malformed key:" prefix discipline.
- **TypeScript (PR-I.1d):** module-private `let _instance` (ESM-strict-mode singleton); `hasForeignType` filter relaxed in JSON-side `isActive` gate (UEBA gate intentionally intact pending UEBA hook).
- **Java (PR-I.1b):** two-files-per-foreign emission (interface + Host class) per Java's "one public top-level type per file" rule.
