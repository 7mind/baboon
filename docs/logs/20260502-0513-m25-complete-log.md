## Cross-cutting architectural notes locked

- **M25-N01** — Indexed-mode UEBA per-field block emission must scope per-field locals (use Kotlin `run { … }` / Scala `locally { … }` / Rust naked `{ … }`). Bare `{ … }` is a lambda; bare statements collide on `val before/after/length`. Closed in PR-25.1.
- **M25-N02** — Cross-runtime conversion registry must key on `(fromTypeId, toTypeId)`. Currently violated in Java (PR-17-D05, deferred); fixed in Dart (PR-25.3). Java parity tracked.
- **M25-N03** — Python facade `json_to_ueba_bytes` interface contract: stringly-typed JSON codecs (pydantic). Round-3 amendment: facade must be tolerant of both shapes (parsed value OR string) for fixture compatibility — stop-gap; permanent fix (single canonical shape) tracked.

## Lessons for future M-cycles

1. **Cross-runtime parity audit before runtime-template semantics flip.** PR-25.4-R01 caught a TS-only divergence; the round-1 executor's reasoning ("codegen always emits a real value") was correct in isolation but missed peer-runtime convention.
2. **Stub codecs in test scaffolding should mirror real-codegen contracts.** PR-25.2 surfaced an iceberg: stub used wrong contract → fixture used different wrong contract → facade had to become tolerant. Mismatched stubs mask bugs in the system under test.
3. **Worktree friction is structural, not incidental.** mdl resolves project-root via `.git`-as-directory, but worktrees use `.git`-as-file. Executors had to mirror edits into main checkout. Same friction observed in M24 Phase 1. Future M-cycles should either (a) accept the mirror-into-main pattern as normal, or (b) fix mudyla's `find_project_root` to handle `.git`-as-file.
