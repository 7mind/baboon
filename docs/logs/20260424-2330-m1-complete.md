### What M1 delivers

End-to-end, the Baboon compiler front-end can now parse, type, and validate `.baboon` models containing `any` fields in all 6 DSL variants:

- `any` → unknown structure from any domain
- `any[domain:this]` → unknown type, current domain, any version
- `any[domain:current]` → unknown type, current version
- `any[T]` → known type T, any domain
- `any[domain:this, T]` → known type T, current domain any version
- `any[domain:current, T]` → fully statically resolved

The typer produces `TypeRef.Any(variant, underlying)`. The validator enforces: (a) underlying must be a user-defined DTO/ADT/Enum with `: derived[ueba]`; (b) `any` cannot be a map key; (c) `any` cannot be a set element.

No code generation yet — every translator has a placeholder `AnyPlaceholder.notSupportedYet(site)` for the `TypeRef.Any` case. LSP hover and `BaboonSchemeRenderer` render readable DSL placeholder strings instead of throwing, so editor tooling works.

## Key surprises / lessons from this session

1. **`TypeRef` cascade surface**: 64 files, not the 9 the plan listed. Plans need to grep exhaustive-match sites before committing to "adding a new sealed case" as a scope.
2. **`mdl :fmt` hazard**: running the whole-tree formatter picks up drift in unrelated files. Executors should use `scalafmt` or `sbt scalafmtOnly <files>` on changed files only.
3. **Parser-typer split for bare `any`**: to preserve back-compat for user types named `any`, bare `any` parses as `Simple("any", Nil)` and only becomes `TypeRef.Any(Global, None)` in the typer. Two entry points into `TypeRef.Any` — documented in spec + plan.
4. **JS bridge has a separate `VerificationIssue` match site** (`BaboonJS.scala:1537`) — easy to miss; future issue-type additions should grep both JVM and JS bridges.
5. **LSP fixture tree walks pick up bad fixtures** — negative-case `.baboon` files in the test resources must be explicitly filtered out of any loader that walks the whole tree.
6. **JSON envelope key collision** with `BaboonTypeMeta` drove the rename to `$ak`/`$ad`/`$av`/`$at`/`$c`. Future schema-side PRs should be aware of this prefix convention.
7. **Meta-kind byte table had an arithmetic error in spec v0** (variants A = 0x07 not 0x05, D1 = 0x06 not 0x04). Caught during planning.
8. **fastparse `|` can't backtrack past consumed input** — critical parser-design lesson from PR 1.1. Correct pattern: parse atomically, then branch on text.
