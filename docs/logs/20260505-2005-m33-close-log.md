## Original orchestrator brief

> Allow a structural-inheritance arm in a `data` / `contract` body to be a
> template instantiation (e.g. `+ MyGeneric[i32]`, `- MyGeneric[i32]`,
> `^ MyGeneric[i32]`). Today the parser only accepts a `ScopedRef` (a name
> path) for the three operators; M33 widens the grammar and the typer
> pipeline so the head can be `Template[T1, ‚Ä¶, Tn]`. Instantiation goes
> through the same monomorphisation machinery that M29 uses for alias-RHS
> instantiation; the resulting concrete shape is then handed to the existing
> structural-composition pipeline. Codegen never sees a template (M29
> invariant preserved).

## PR-33.2 ‚Äî Typer (the architectural meat)

Inline substitution at typer-early time per decision ¬ß3.b Option I. The
template body's substituted members are inlined into the receiving DTO's
member list verbatim ‚Äî no transient `DomainMember.User`, no synthetic
identifier exposed. Codegen never sees a template (M29 invariant
preserved end-to-end).

**Architectural surprise ‚Äî `^` needed an M3 fallback.** Pure inline has
no `ScopedRef` for the existing `IntersectionDef`-translator path under
the ¬ß3.b "no synthetic id" constraint. Smallest carrier: a new sealed
branch `RawDtoMember.IntersectionFields(fields, meta)` that lives in the
raw AST after lowering and has explicit translator handling in
`BaboonTranslator.intersectionLimiters`. Documented at
`RawDtoMember.scala:21-39` and `TemplateInstantiator.scala:147-158`.

**Defect D02 ‚Äî `-`/`^` cannot accept template bodies that themselves use
structural composition from concrete types.** A template body containing
`+ ConcreteBase` would have `Base`'s contributions silently lost. New
typer issue `TemplateBodyNotFlatForRemoval` fires at lowering time, with
the M29 3-site exhaustive-match bundling pattern (DiagnosticsProvider /
WorkspaceState / BaboonJS) plus printer arm. One TyperIssue case, three
exhaustive-match site updates, one printer arm ‚Äî bundled per the M29
close-out playbook.

**Defect D05 ‚Äî bare-template detection had to move pre-toposort.**
`BaboonTyper.order()` runs first; without an early bare-template check,
`hardDepsOfRawDefn` returns a bare template ref that `resolveScopedRef`
fails on with confusing `NameNotFound`. Solution: new
`validateNoBareTemplateRefs` validator on the `TemplateInstantiator`
trait, called from `BaboonTyper.scala:419-425` BEFORE `order()`.
`hardDepsOfRawDefn` correspondingly tightened to skip `args.isDefined`
arms (validator clears them before `order()`).

**Recursion guard.** Hard-coded depth 32 + cycle-set keyed by
`(receivingName, templateName, argTupleKey)`. D04 attempt at constructor-
tunable failed because distage cannot fill an unbound `Int` from a Scala
default ‚Äî reverted to constant. Self-deepening `Deep[lst[T]]` exercises
depth-limit; mutual cycles exercise cycle-set.

**Tests.** New file `M33StructuralTemplateInstantiationTest.scala` (14
initial cases). `sbt baboonJVM/test` 539/540 green (one pre-existing
`RTCodecTest` failure, unrelated, reproduces on stashed HEAD).

## PR-33.4 ‚Äî Cycle / recursive-substitution detection

Bulk landed in PR-33.2/PR-33.3. Residual: empty-intersection edge case
flagged in PR-33.2 round-2 advisory. When `^ Template[T]`'s substituted
body is empty, the existing `BaboonTranslator.scala:319`
`if (intersectionSet.isEmpty)` short-circuit silently turned `^ Empty[T]`
into a no-op pass-through. Now fails at lowering with
`TemplateBodyNotFlatForRemoval` carrying sentinel
`offendingMemberKind = "empty body"` (`TemplateInstantiator.scala:553-572`).
Printer branches on the sentinel: "the substituted body is empty
(intersection over an empty field set would be a silent no-op; caught at
lowering time‚Ä¶)".

Cycle-key canonicalisation hardened from `RawTypeRef.toString` to the
explicit `RawTypeRef.render` form (`TemplateInstantiator.scala:449`) ‚Äî
identical strings today, immune to non-canonical-field drift.

Regression-guard tests added for `+ Empty[i32]` (positive no-op pass) and
`- Empty[i32]` (positive no-op pass with comment naming deferred
[PR-33.4-D01] silent-dedup-style asymmetry).

`M33StructuralTemplateInstantiationTest` 15 ‚Üí 17 tests. `sbt baboonJVM/test`
550/550 green.

**No new TyperIssue case.** Reused `TemplateBodyNotFlatForRemoval` with
sentinel string to avoid the M29 3-site exhaustive-match cost; clear
sentinel-aware printer branch keeps the rendered diagnostic readable.
