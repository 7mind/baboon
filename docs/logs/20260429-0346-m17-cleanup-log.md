## Outcomes

### Wontfix closures
- **PR-35-D08** (runtime codec error enrichment) — closed wontfix. Runtime codec only sees the wire form on the decode path; "source name" doesn't exist at the call site. Adding it would require either plumbing source-name through every codec call or embedding a source→wire map per enum at codec construction. Disproportionate to diagnostic value.
- **PR-40-D08** (sbt resource-cache stale-bug) — closed wontfix. Known interaction between sbt's incremental compilation and Scala 3 macro inline expansion: macro-embedded resource bytes are baked at compile time and the sbt change-tracker does not register the macro's transitive resource dependency. Workarounds add build complexity for a dev-only ergonomic issue. The `sbt clean` requirement is acceptable when editing `baboon-runtime/*` resources.

## Process notes

- **Planning-subagent harness limitation surfaced.** All four planning subagents (Plan + 3 general-purpose) returned plans as text rather than writing to disk. Their internal harness blocks file writes. The orchestrator persisted each plan to disk afterwards, fixing HTML entity encoding (`&amp;`/`&gt;`/`&lt;`) along the way. Process improvement for next session: pre-create the target plan files (even empty) so subagents have an editable artifact, OR explicitly tell subagents to use a different return mechanism.
- **PR-42 reclassification by planner.** The original defect entry [PR-39-D03] claimed the Scala UEBA enum codec emitted "first arm at column 0, subsequent at column 2 — opposite-direction misalignment from Dart/Java/Kotlin". The M17 planning subagent inspected current generated output and found this claim was **wrong** — the surrounding `genCodec` wrap normalised whitespace, so generated output is correct. The two `genEnumBodies` templates were merely the only `shift(2)` invocations in `ScUEBACodecGenerator.scala` that didn't follow the inline `${...joinN().shift(2).trim}` form used by the other 19 sites in the same file. PR-42 was therefore reclassified as a **refactor for symmetry**, not a bug fix; commit message says so. Snapshot diff verified byte-identical regenerated output.
- **PR-43 first-pass deviation surfaced two majors via adversarial review.** The first executor (a) tagged TS/Py helpers with `CompilerProduct.Runtime` instead of `Test` to work around an import-path resolution issue (M1), (b) relaxed the strict `target/ + *-stub` sentinel to just `*-stub` because `mdl :test-scala-regular` without `:test-cs-regular` first found `target/` missing (M2), and (c) gated the bootstrap assertion to env-var-only (M3) — which together reintroduced the silent-skip class of defect that PR-40-D02 was specifically written to prevent. Adversarial review caught all three; fix round implemented a layered sentinel (STRICT first, NAMED `test-regular|test-wrapped + *-stub` fallback) plus an anchor/fixtureRoot split that preserves the unconditional bootstrap assertion. The second adversarial review then surfaced that PR-40's Dt/Sw helpers were left with the strict-only logic and would now diverge from PR-43's helpers — second fix round retro-applied the layered logic to Dt/Sw for symmetry.
- **Two-round adversarial review on PR-43 was load-bearing.** Without round 1: silent-skip regression on Sc/Ts/Rs/Py. Without round 2: cross-helper inconsistency between Dt/Sw (strict-only) and the new ones. Both rounds caught real bugs the executor missed.
