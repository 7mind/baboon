## Critical follow-up: ADT-arm parser/typer doc gap

[PR-30.4-D02] is a real defect tracked as a PR-30.2 / PR-30.3
follow-up:
- Parser at `DefAdt.adtMember` wraps each ADT arm DTO in
  `meta.withMeta(...)`, which calls `prefixDocs` and captures the
  doc.
- BUT `dtoEnclosed` (the body of the wrapper) itself uses
  `meta.member(kw.data, ...)` which runs `withMeta` AGAIN. By that
  point, the input has been advanced past the doc, so the inner
  `RawDto.meta.docs.prefix` is `None`.
- Downstream `convertDto` builds `DomainMember.User(..., dto.meta)`
  using the inner meta only, so the wrapper's
  `RawAdtMemberDto.meta.docs.prefix` is silently dropped.
- Fix: pick (a) in `convertDto` for ADT-arm DTOs prefer
  `RawAdtMemberDto.meta` over the inner `dto.meta` (or merge), OR
  (b) in `DefAdt.adtMember` flatten so `dtoEnclosed` does not
  re-consume `prefixDocs`.
- This must be fixed **before PR-30.15** (cross-language smoke
  fixture) so per-backend tests can exercise ADT-arm-with-docs.

The fix is small and parallels PR-30.3-D01's restructure.

## Constraints future work must respect

1. Backends consume `Field.docs.prefix.cleaned` etc. directly; do
   NOT re-clean. `Docs.cleaned` is the canonical input.
2. Layered drop means backends never see docs at non-position
   carriers (enum values, ADT inheritance arms, namespaces, foreign
   per-language entries).
3. Schema-id digest stability is structural by construction; any
   future refactor must keep the projection in
   `BaboonEnquiries.shallowId` explicit and excluding new fields.
4. `BaboonWhitespace` is the canonical Baboon parser whitespace —
   any new parser site MUST use it, NOT `fastparse.ScalaWhitespace`.
5. PR-30.4 establishes the per-backend template. Subsequent backend
   PRs should mirror its file-touch list:
   `translator/<lang>/{*DefnTranslator,*TreeTools}.scala` + new
   emission test + new fixture.
