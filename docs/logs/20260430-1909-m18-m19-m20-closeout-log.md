## Key design decisions surfaced during execution

1. **PR-57e Rust stub `#[path]` routing** — `test/conv-test-rs/src/lib.rs` is hand-curated (not generator-produced); each new top-level runtime module needs manual entry. Possible future hygiene PR: auto-route via `include!("generated/lib.rs")`.

2. **PR-59 Q-FU-1 derivation-parity recursion** — round-1 review caught that the validator only checked derivation parity on the immediate map-key user type. Round-2 fix: thread the visited-user-type chain through `isEligibleKey`'s return value; apply parity check at every node. Without this, M19's validator-catches-before-codegen guarantee (Q-M19-8) was incomplete.

3. **PR-60 TS decoder string→primitive parsing** — the `as number` type-system cast doesn't parse strings at runtime. Round-2 fix: explicit `parseInt`/`parseFloat`/`BigInt(...)` for map keys.

4. **PR-60 Python non-any-bearing path** — Pydantic `model_dump_json` doesn't reliably use `__str__` for user-typed dict keys; decode definitively fails without `@field_validator`. Round-2 fix: detect `dtoHasUserKeyMapField` and route through the explicit walker.

5. **PR-61 Rust serde plumbing** — Rust's JSON codec uses derived `Serialize`. Approach (b): per-K adapter module via `#[serde(with = "...")]` field attribute. `BTreeMap<String, V>` intermediate ensures deterministic key ordering for cross-language byte-identity.

6. **PR-62 ScopedRef ambiguity** — round-1 review BLOCKER on namespace-qualified ADTs. The "1 segment → All / 2+ segments → Branch" dispatch silently misparsed `+ pkg.sub.MyAdt` (3 segments) as IncludeBranch. Round-2 collapsed 5 variants to 3 (Include/Exclude/Intersect each with unsplit ScopedRef); All-vs-Branch resolution deferred to PR-63's typer-early pass which has full namespace context.

7. **PR-63 `CrossVersionAdtInclusion` is dead code** — `ScopeSupport.resolveTypeId` always uses the current domain's pkg, so no baboon source can produce a TypeId.User with a different pkg within a single domain's expansion. Defensive check kept; documented in Scaladoc.

8. **PR-64 plan §5 strict-reading deviation** — manual→sugared rewrite lands in `deepModified` not `unmodified` due to `deepSchemaRepr` not sorting ADT branches. Operationally harmless (`BaboonRules.sameLocalStruct = unmodified || deepChanged` treats both identically). Deferred to hygiene PR.
