## What M4 delivers

End-to-end, the Rust target now emits real codec code for `any` fields in all six DSL variants. Both UEBA (binary) and JSON (serde) codecs are generated. The runtime ADT (`AnyOpaque = AnyOpaqueUeba(meta, Vec<u8>) | AnyOpaqueJson(meta, serde_json::Value)`) matches the Scala/C# shape with Rust-idiomatic content equality (`Vec<u8>` and `serde_json::Value` `PartialEq` are content-wise by default — no manual override needed, unlike Scala/C#).

Specific capabilities mirroring M2/M3:
- **Round-trip both directions**: encode an `AnyOpaque` in its native format, decode back, content equality holds.
- **Cross-format conversion via facade**: `AnyOpaque::Ueba` arriving at JSON-encoder context can convert to JSON via `facade.ueba_to_json(meta, bytes, statics)` (and symmetric).
- **Static fallback for D variants**: codec generator threads field-declaration statics (current domain, current version, underlying typeid) into cross-convert calls.
- **`facade.decode_any`**: resolves typed value from `AnyOpaque` via registry lookup.
- **Forward-compat meta-length skipping**: decoders honor on-wire `meta-length` and skip extension bytes via `read_exact` on a buffer.
- **Auto-generated round-trip tests** via branch-matching fixtures: `test_holder_json_codec` and `test_holder_ueba_codec` both pass.
- **Cross-language wire compat with Scala/C#**: `bin_tools::write_string` is byte-equivalent to Scala/C#'s ULEB128 + UTF-8.
- **`serde_json/preserve_order`** enabled so JSON envelope key ordering matches Scala/C# insertion order.

## Key surprises / lessons from this session

1. **PR 4.1 round-1 executor's "full Scala-facade port" claim was inaccurate**. Six entry points and the conversion protocol were stubs. Reviewer caught the gap; round-2 wrote ~400 missing lines including `decode_from_bin`, `decode_from_json`, `convert<>`, `convert_typed<TTo>`, ADT-aware `BaboonTypeMeta::from`, and the full `BaboonAnyConversion`/`BaboonAnyConversions` trait surface. Lesson: when porting a multi-language facade, verify entry-point parity surface-by-surface — don't accept "full port" as a black-box claim.
2. **`BaboonCodecContext` as enum vs struct**: Rust's existing 3-variant enum (Default/Indexed/Compact) had to grow a 4th variant `WithFacade { use_indices, facade }` rather than convert to struct. Pattern-match exhaustiveness checks surfaced every site needing the new arm.
3. **Custom serde `Serialize`/`Deserialize` for `AnyOpaque`**: auto-derive would emit `{"Ueba":...}` shape, not the locked envelope. Custom impls emit `{"$ak":..., "$c":...}` for `Json` branch and error on `Ueba` branch (caller must convert via facade first; codec-generator-emitted code routes through `facade.ueba_to_json` before serde).
4. **`Vec<u8>` and `serde_json::Value` PartialEq are content-wise in Rust** — no manual override needed (PR-05-D08 and PR-08-D06 lessons are non-issues here, unlike Scala/C#). Rust's stdlib derives the right semantics by default.
5. **Type erasure via `Box<dyn BaboonGeneratedDyn>`**: Rust trait objects + dyn dispatch differ from Scala/C# class hierarchies. The `into_any` method allows downcasting via `Box<dyn Any>::downcast` for `convert_typed<TTo>`.
6. **Wire-format negative-i32 trap** (PR-12-D01): Rust's `i32 as usize` cast on a negative value produces a giant unsigned, panicking on `4 + meta_length` arithmetic in debug or wrapping in release. Sibling C#/Scala explicitly check for negative before casting. Future Rust port-style fixes need similar discipline.
7. **`Eq`/`Ord` derive suppression on any-bearing DTOs** (PR 4.2 scope expansion): `serde_json::Value` doesn't impl `Ord`, so codegen emits non-compiling `derive(Ord)` on Holder. Mirrors the existing `hasDirectFloat` axis. Future Rust language additions of types lacking `Ord` should follow the same pattern.
8. **Runtime `?Sized` follow-up** (also PR 4.2): PR 4.1 oversight — `read_bin_with_length` needed `?Sized` bound to accept `&mut dyn Read` from codec-emitted code. Future Rust runtime additions targeted by codec emission should default to `?Sized`.
9. **JSON key-ordering trap with serde_json** (PR-11-D05): default `serde_json::Map` is `BTreeMap` (alphabetical); enabling `preserve_order` feature flag is required for cross-language wire compat with Scala/C# insertion-order semantics.
10. **No per-domain auto-generated facade subclass yet in Rust**: tests construct facades manually via inline trait-object adapters wrapping codec singletons. Mirrors the `SampleV1`/`SampleBinCodec` precedent. C# auto-generates `My.Ok.BaboonCodecsJson.Instance`; Rust does not (yet). Future enhancement, not blocking.
