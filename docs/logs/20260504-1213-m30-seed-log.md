## Why these four questions block PR-30.1 specifically

PR-30.1 is the spec doc `docs/spec/docstrings.md`. The spec freezes
surface syntax. Q1 picks the surface semantics for the M29-cross-link
in the spec; Q2 picks the Python emission syntax that the spec
documents per-backend; Q3 picks the parser's accumulation rule which
the spec must publish so users can compose docs predictably; Q4 picks
where the cleanup rule lives, which determines the spec's "doc body
content" definition.

Without those four answers the spec would either ship under-specified
(forcing later breaking changes) or ship under unilateral planner
choice (CLAUDE.md §3 — multiple interpretations exist; do not pick
silently). The loop correctly stops here.
