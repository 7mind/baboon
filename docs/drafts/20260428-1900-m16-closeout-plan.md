## Phase 1 — Per-item complexity analysis

### Item 1 — PR-32-D03: Validator predicate

**Status:** Design-corrected (was: simple flip → actually: rename + recompute + flip).

`BaboonValidator.scala:597,607` currently:
```scala
val missingDiffs = nextIds.diff(diffIds)
F.when(missingDiffs.isEmpty)(F.fail(MissingEvoDiff))
```

`BaboonComparator` builds `diffs` only from `changed` and `validRenames` — every unmodified or newly-added type is absent from `diffIds`. So `missingDiffs` is non-empty in basically every realistic model. The current predicate `isEmpty` only fires for the degenerate case where literally every type has a diff entry — which is what PR-34's 2-type test happened to be (both types renamed).

**User's stated intent:** "fail if next version has type diffs which are not bound to an existing type id, a basic sanity check — so there should be non-empty".

That semantically describes `diffIds.diff(nextIds)` (orphan diffs — diffs whose type isn't in `next`). The current variable is the OPPOSITE direction.

**Fix:** rename `missingDiffs` → `orphanDiffs`; recompute as `diffIds.diff(nextIds)`; predicate `nonEmpty`. Then remove the placeholder `Stable` DTO from `ScEnumConversionTest.scala`. The `Stable` was masking the inverted-predicate-firing bug; with orphan-diffs semantics, a 2-type renamed-only model is fine (no diffs are orphaned).

### Item 8 — Dart cross-language path-coupling — REVISED SCOPE

**Status:** 2-language fix (Dart + Swift), not 1.

`DtCodecTestsTranslator.scala:212-230` AND `SwCodecTestsTranslator.scala` (around lines 202, 232, 247, 277) both emit relative paths like `../../../../../target/<lang>/json-default/$typeId.json` (5 dot-dots, escapes the repo). Other languages either don't have cross-language reads or use shorter paths.

**Fix candidates:**
- (a) Env var `BABOON_CROSS_LANG_FIXTURE_ROOT` read at test time, with a sensible default

- (c) Bake a CompilerOption-injected absolute path into generated tests

Recommended: (a) with a `.git`-walk-up fallback. Touches 2 generators + the corresponding language-side fixture-write paths to ensure both producers and consumers agree.

**Risk:** must verify cross-language tests don't silently skip via `File.existsSync()` if the path is wrong. Add an assertion that at least one cross-lang fixture is read.
