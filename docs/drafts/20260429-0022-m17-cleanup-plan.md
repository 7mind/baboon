## PR-41 — `BaboonValidator` filter symmetry (closes PR-36-D03)

### Scope
- File: `baboon-compiler/src/main/scala/io/septimalmind/baboon/validator/BaboonValidator.scala`
- Lines: 595–599 (the `validateEvo` method's id-set construction).

### Root cause
Two adjacent set constructions use asymmetric type filters:
```scala
val nextIds = next.defs.meta.nodes.keySet                                     // line 595 — all TypeIds incl Builtin
val prevIds = prev.defs.meta.nodes.collect {                                  // lines 597–599 — User only
  case (id: TypeId.User, _) => id: TypeId
}.toSet
```
Both sets are then consumed by `diffIds.diff(nextIds).diff(prevIds)` (line 603) and by the conversion-coverage diff at lines 604–606. `BaboonComparator.scala:219` asserts `changed.forall(_.isInstanceOf[TypeId.User])`, so all `diffIds` are `TypeId.User` in practice — the asymmetric filter is harmless today but reads confusingly and is a latent footgun if a future change introduces a Builtin diff entry.

### Decision: filter both to `TypeId.User`

**Recommended option:** apply the same `.collect` to `nextIds`. Rationale:
1. The semantically meaningful comparison is "User type renamed/changed/added/removed". Builtins never participate. Filtering both narrows the contract explicitly, matching `BaboonComparator.scala:219`'s assertion.
2. The conversion-coverage check at lines 605–606 (`prevIds.diff(conversionIds)`, `conversionIds.diff(prevIds)`) is symmetrical only if `prevIds` and `conversionIds` are both User-only; conversions are over user types by construction. Keeping `prevIds` filtered matches the existing semantics.
3. Dropping the filter on `prevIds` would *expand* the contract to allow Builtin types in conversions, which is semantically wrong.

### Exact change
Replace lines 595–599 with:
```scala
val nextIds = next.defs.meta.nodes.collect {
  case (id: TypeId.User, _) => id: TypeId
}.toSet
val diffIds = diff.diffs.keySet
val prevIds = prev.defs.meta.nodes.collect {
  case (id: TypeId.User, _) => id: TypeId
}.toSet
```

### Behaviour change
None. `diffIds` are all `TypeId.User` per `BaboonComparator.scala:219`, so `diffIds.diff(nextIds)` produces the same result whether `nextIds` includes Builtins or not.

### Success criteria
1. `sbt baboonJVM/compile` — clean.
2. `sbt baboonJVM/test` — 193/193 pass.
3. `mdl :test-gen-regular-adt :test-gen-wrapped-adt` — both PASS.
4. `git diff -- target/test-regular target/test-wrapped` shows zero changes (semantic-equivalence check).

### Risks
- **Negligible.** Pure refactor.

## PR-42 — `ScUEBACodecGenerator` enum template restructure (closes PR-39-D03)

### Scope
- File: `baboon-compiler/src/main/scala/io/septimalmind/baboon/translator/scl/ScUEBACodecGenerator.scala`
- Lines: 309–321 (the `genEnumBodies` method's two return-tuple template literals).

### Root cause analysis (correction to defect text)

The defect entry [PR-39-D03] at `defects.md:1606–1611` claims the Scala UEBA enum codec emits "first arm at column 0, subsequent at column 2 — opposite-direction misalignment from Dart/Java/Kotlin." **Investigation contradicts that claim.**

Inspection of current generated output:
- `target/test-regular/sc-stub/src/main/scala/generated-main/testpkg/pkg0/T1_E1.scala` lines 67–70 (2-member enum)
- `target/test-regular/sc-stub/src/main/scala/generated-main/testpkg/pkg0/T1_E2.scala` lines 70–74 (3-member enum)

All show **uniform 8-space indentation** for all `case` arms — no misalignment is observable today. The template at `ScUEBACodecGenerator.scala:309–321` happens to produce correct output because the surrounding `genCodec` wrap re-shifts and re-trims uniformly.

**However, the deferred-fix suggestion is still valuable** for:
1. **Cross-backend symmetry**: Dart/Java/Kotlin all use the `q"""...\n   |  ${branches.map(_._1).joinN().shift(2).trim}\n   |..."""` pattern. Scala uses an inconsistent shape (`|${...shift(2)}` on its own line, no `.trim`).
2. **Maintenance hazard**: future changes to either pattern risk diverging output silently. The current Scala template only happens to align by coincidence of the enclosing template's `.shift(2).trim`.

PR-42 is therefore a **refactor for symmetry**, not a bug fix. The PR description and commit message must say so.

### Current code (Scala, before fix)
```scala
// ScUEBACodecGenerator.scala:309-321
(
  q"""value match {
     |${branches.map(_._1).joinN().shift(2)}
     |}
     """.stripMargin,
  q"""val asByte = wire.readByte()
     |
     |asByte match {
     |${branches.map(_._2).joinN().shift(2)}
     |  case _ => Left(new $genericException(s"Cannot decode {wire} to ${name.name}: no matching value"))
     |}
     |""".stripMargin,
)
```

### Reference pattern (Java, after PR-39 fix)
```scala
// JvUEBACodecGenerator.scala:248-260
(
  q"""switch (value) {
     |  ${branches.map(_._1).joinN().shift(2).trim}
     |}
     """.stripMargin,
  ...
)
```

### Proposed code (Scala, after fix)
```scala
(
  q"""value match {
     |  ${branches.map(_._1).joinN().shift(2).trim}
     |}
     """.stripMargin,
  q"""val asByte = wire.readByte()
     |
     |asByte match {
     |  ${branches.map(_._2).joinN().shift(2).trim}
     |  case _ => Left(new $genericException(s"Cannot decode {wire} to ${name.name}: no matching value"))
     |}
     |""".stripMargin,
)
```

Diff is purely:
- Move `${...}` from its own line (preceded by `|`) to inline after `|  ` (2-space margin).
- Append `.trim` after `.shift(2)`.

### Behaviour change

Expected: byte-for-byte identical generated output. Verify empirically by regenerating and `diff -u` against pre-fix.

### Success criteria
1. `sbt baboonJVM/compile` clean.
2. `mdl :test-gen-regular-adt :test-gen-wrapped-adt` PASS.
3. Generated `T1_E1*.scala`, `T1_E2.scala`, `VaultState.scala` show uniform indentation.
4. `mdl :test-scala-regular :test-scala-wrapped` PASS.
5. Pre-fix vs post-fix generated output diff is empty.

### Verification commands
```bash

```

### Risks
- **Low.** Mechanical substitution; only risk is a subtle whitespace edge case — verifiable via empty diff.

## PR-43 — Generalize `crossLanguageFixturePath` helper across applicable backends (closes PR-40-D07)

### Per-backend audit

| Backend | File | Cross-lang reads | Reads from cs/ only? |
|---|---|---|---|
| Scala | `ScCodecTestsTranslator.scala` | lines 91 (json), 126 (ueba) | YES |
| Kotlin (incl. KMP) | `KtCodecTestsTranslator.scala` | **none** | N/A |
| Java | `JvCodecTestsTranslator.scala` | **none** | N/A |
| TypeScript | `TsCodecTestsTranslator.scala` | lines 102 (json), 158 (ueba) | YES |
| Rust | `RsCodecTestsTranslator.scala` | line 91 (json only; no ueba cross-lang read) | YES |
| Python | `PyCodecTestTranslator.scala` (singular!) | lines 70 (json), 101 (ueba) | YES |

**The user's claim "they happen to work because they only read from `target/cs/...`" is verified for Sc/Ts/Rs/Py.** Each uses a relative path resolving to `<isolation-root>/target/cs/...` because tests run from cwd `<isolation-root>/<lang>-stub` via `pushd` in `mdl` actions.

**Kotlin and Java are explicitly excluded from PR-43 scope.** No cross-language reads to fix.

### Decision: one fat PR-43, not split per-backend

Each backend's emitter change is mechanical and ~50 LoC. ~200 LoC total. Bundling keeps the design pattern reviewable in one place. Mitigation if one backend regresses during review: revert that one's changes within the PR.

### Generic design pattern (mirrors PR-40 exactly)

For each in-scope backend:

1. **Add a runtime resource file** `cross_language_fixture_path.<ext>` with three exports:
   - `crossLanguageFixturePath(lang, type, format)` — walks up to find sentinel directory (subdir `target/` AND a sibling subdir matching `*-stub`); env var `BABOON_CROSS_LANG_FIXTURE_ROOT` override; throws on no-match.
   - `crossLanguageFixtureRoot()` — exposes resolved root.
   - `assertCrossLanguageFixtureRootExists()` — bootstrap-time loud-fail.

2. **Add `sharedTestHelper()` method** to each `*BaboonTranslator.scala`, mirroring `DtBaboonTranslator.scala:160–176`. Conditional on `target.output.products.contains(CompilerProduct.Test)`, emits the helper file via `BaboonRuntimeResources.read(...)` with `doNotModify = true`.

3. **Rewrite codec test translator** to:
   - Replace hardcoded relative paths with `crossLanguageFixturePath(...)` calls.
   - Inject a setup hook calling `assertCrossLanguageFixtureRootExists()` at test-class bootstrap.
   - **Retain** existing per-test `existsSync()`/`fileExists()` skip (peer language hasn't run yet — legitimate skip).

### Per-backend specifics

#### Scala
- **Resource:** `baboon-compiler/src/main/resources/baboon-runtime/scala/CrossLanguageFixturePath.scala`, package `baboon.runtime.shared`. Use `java.nio.file` for cwd, `Files.list` walk-up, env var via `sys.env.get`, throw `IllegalStateException` on no-match.
- **Translator wrapper:** `ScBaboonTranslator.scala` add `sharedTestHelper()` analog.
- **Codec test rewrite:** `ScCodecTestsTranslator.scala` lines 89–100, 124–134 replace path expressions with helper calls. Mix in ScalaTest `BeforeAndAfterAll` and override `beforeAll` to call the assertion. Retain `assume(f.exists())` per-test skip.

#### TypeScript
- **Resource:** `baboon-compiler/src/main/resources/baboon-runtime/typescript/CrossLanguageFixturePath.ts`. Use `process.cwd()`, `fs.readdirSync` walk-up, `process.env.BABOON_CROSS_LANG_FIXTURE_ROOT`, throw on no-match.
- **Translator wrapper:** `TsBaboonTranslator.scala` add helper analog. Existing TS translator already uses `BaboonRuntimeResources.read(...)` patterns.
- **Codec test rewrite:** `TsCodecTestsTranslator.scala` lines 102, 158 replace template literals. Bootstrap hook: vitest `beforeAll(() => { assertCrossLanguageFixtureRootExists() })` at top of each test file.
- **NB:** cs writes `.uebin` UEBA extension; readers must match. Do not change extensions.

#### Rust
- **Resource:** `baboon-compiler/src/main/resources/baboon-runtime/rust/cross_language_fixture_path.rs`. Use `std::env::current_dir()`, `std::fs::read_dir` walk-up, `std::env::var`, `panic!` on no-match.
- **Translator wrapper:** `RsBaboonTranslator.scala` add helper analog.
- **Codec test rewrite:** `RsCodecTestsTranslator.scala` line 91. Bootstrap: no global setUp in std test harness — use `std::sync::Once` cached lazy init inside the helper PLUS explicit `assert_cross_language_fixture_root_exists()` as first statement of every generated test function.

#### Python
- **Resource:** `baboon-compiler/src/main/resources/baboon-runtime/python/cross_language_fixture_path.py`. Use `os.getcwd()`, `os.scandir` walk-up, `os.environ.get`, raise `RuntimeError`.
- **Translator wrapper:** `PyBaboonTranslator.scala` add helper analog.
- **Codec test rewrite:** `PyCodecTestTranslator.scala` lines 70, 101. unittest-style → inject `def setUpClass(cls): assert_cross_language_fixture_root_exists()` at class scope.
